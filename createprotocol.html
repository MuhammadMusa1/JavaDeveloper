Зачем знать протоколы?

  • Финансовые протоколы: Единый язык для банков и платёжных систем.
  • Аналогия: Как общий язык для поваров на кухне.
  • Необходимы для финтеха: Как SQL для баз данных.
  • Пример: Ориёнбанк отправляет платёж за границу.

Слайд 3

Какие протоколы рассмотрим?

  • ISO 8583: Для карточных транзакций.
  • SWIFT: Международные платежи (MT/MX).
  • ISO 20022: Универсальный стандарт (XML).
  • SOAP/REST: API для финтеха.
  • MQ: Надёжный обмен сообщениями.

Визуализация: Схема: “Экосистема финансов”.

Слайд 4

ISO 8583: Карточные транзакции

  • Стандарт для банкоматов и POS-терминалов.
  • Бинарный формат: Компактный, но сложный.
  • Пример: Платёж картой в Ориёнбанке.
  • Аналогия: Как заказ в ресторане: чёткие поля.
Слайд 5

Структура ISO 8583

  • MTI: Тип сообщения (например, 0200 — запрос).
  • Bitmap: Указывает, какие поля включены.
  • Data Elements: Данные (сумма, номер карты).
MTI: 0200
Bitmap: 7F22000000000000
DE2: 1234567890123456 (номер карты)
DE4: 000000010000 (1000 TJS)

Визуализация: Схема: “Сообщение ISO 8583”.

Слайд 6

Код: ISO 8583 с j8583

import com.solab.iso8583.IsoMessage;

IsoMessage msg = new IsoMessage();
msg.setType(0x200); // Запрос авторизации
msg.setField(2, "1234567890123456"); // Номер карты
msg.setField(4, "1000"); // Сумма
msg.setField(49, "TJS"); // Валюта

Создаём сообщение для Ориёнбанка.

Слайд 7

Особенности ISO 8583

  • Бинарный: Компактный, но сложный для чтения.
  • Bitmap: Показывает активные поля.
  • Библиотеки: Используйте j8583 или jPOS.

Вопрос: Зачем нужна Bitmap?

Слайд 8

SWIFT: Международные платежи

  • SWIFT: Сеть для банковских переводов.
  • MT: Текстовые сообщения (старый формат).
  • MX: XML-сообщения (новый формат).
  • Пример: Перевод из Ориёнбанка в другой банк.
Слайд 9

Пример: MT и MX

// MT103 (перевод)
:20:TRANS123
:32A:250807USD1000,00
:50K:Ali Sherov
:59:/123456789
Orienbank, Dushanbe

// MX (pain.001, ISO 20022)

    TRANS123
    1000.00
    USD

MT — текст, MX — XML.

Особенности SWIFT

  • MT: Фиксированный текстовый формат.
  • MX: XML, переход к ISO 20022.
  • Безопасность: Высокая надёжность сети.

Слайд 12

ISO 20022: Универсальный стандарт

  • XML-формат для всех финансовых операций.
  • Гибкий и читаемый.
  • Пример: Платёж в Ориёнбанке.
  • Аналогия: Универсальный переводчик.
Слайд 13

Код: ISO 20022 с JAXB

import javax.xml.bind.JAXBContext;

JAXBContext context = JAXBContext.newInstance(PaymentInfo.class);
Unmarshaller unmarshaller = context.createUnmarshaller();
PaymentInfo payment = (PaymentInfo) unmarshaller.unmarshal(new File("payment.xml"));

XML в Java-объект.

Слайд 14

Особенности ISO 20022

  • XML: Легко читать и расширять.
  • Метаданные: Много информации о платеже.
  • Будущее: Заменяет старые стандарты.

Вопрос: Почему ISO 20022 популярен?

Слайд 15

SOAP/REST: API для финтеха

  • REST: Простой, JSON, HTTP.
  • SOAP: XML, строгий, WSDL.
  • Пример: Запрос баланса в Ориёнбанке.
Слайд 16

Код: REST с OkHttp

import okhttp3.OkHttpClient;
import okhttp3.Request;

OkHttpClient client = new OkHttpClient();
Request request = new Request.Builder()
    .url("https://orienbank.tj/api/balance")
    .build();
Response response = client.newCall(request).execute();
System.out.println(response.body().string());

Получаем баланс через REST.

Слайд 17

Особенности SOAP/REST

  • REST: Лёгкий, популярен в финтехе.
  • SOAP: Строгий, для сложных систем.
  • HTTP: Основа для обоих.

Визуализация: Схема: “REST vs SOAP”.

  • Пример: Ориёнбанк отправляет транзакции в очередь.
